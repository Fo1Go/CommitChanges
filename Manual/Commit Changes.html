Module divided on 3 sections: some notes, a screen with filenames and a console button. There are repository name, contributer name and day of week on piece of paper. The screen with filenames below, which count can be from 5 to 8, buttons which switch filenames in cycle, a LED, which can be on of 3 states: red, yellow, green. On the bottom of the module there is the button, when pressed opens console.

In order to solve the module, you must set each file correct state and commit them in correct order. You can commit files using console with Git commands (Appendix G1t). Every commit must include exactly one file. After all files commited, you must push changes in remote repository. If you done all correct, module will solve, otherwise handle strike, and show index of first wrong commited file.
Note: After strike, module doesn't reseting. You must do it by yourself.

File value

Look all conditions in table on page 2. If the condition is true, do action at the column "Action".

If the file value is less than zero, make him positive. That is the final file value(FFV). If number of files is even, sort them by his FFV in ascending order, otherwise in descending order. This is the order you must commit files. If you have same FFV for two files or more, first that appears on module is preferred.

File status

Take FFV, multiply on coefficient of initial status (Table on page 1). If number has any decimal, make him integer. Lets name him X. Then do bitwise XOR between X and multiplication of repository coefficient and Day of week coefficeint, modulo 2. This is the file final status.

Repository coefficient

  • Sum up all alphabetical position of letter in repository name.
  • Modulo number symbols multiply by 2.
  • Add 1

Appendix G1t

Our company uses AI for helping you, so you don't need to add in your commands mandatory parameters by yourself.

git add [filename]

Adding file. Setting status 1 when commited. Also you can use file index instead filename, where first file appears on module is one.

git rm [filename]

Removing file. Setting status 0 when commited. Also you can use file index instead filename, where first file appears on module is one.

git commit

Committing changes. Note: in one commit must be exactly one file. If in the commit be more than one file or zero, module will strike immediately.

git push

Pushing all commits in remote repository. While command is inputed, module will check solution. If your solution is correct module will be solved. Otherwise module will strike and show first wrong file.

git status

Показывает текущее состояние репозитория, те файлы которые были добавлены/удалены. Например, если только что произошёл коммит, то эта команда покажет clear, так как от предыдущего коммита и текущего состояния репозитория нет отличий. При выполнении команды git add, покажется файл, который был добавлен. Аналогично работает с git rm, будет показан файл, который был удалён. Команда необходима для проверки того, что вы коммитите.

git reset HEAD

Возвращает репозиторий к состоянию до команды git add/rm. (То есть, до состояния репозитория после коммита)

Возвращает репозиторий к изначальному состоянию. (То есть, при инициализации модуля)
