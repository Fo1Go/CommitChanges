On the Subject of Commit Changes

Should we commit it on Friday?

Module are divided into 3 sections.

  • A paper with notes which consists on it a repository name, contributor name and day of week.
  • The screen with file names below, the number of which can range from 3 up to 5, buttons that switch file names in cycle, and a colored LED, that can be one of two states: red, green. The first file have green color.
  • At the bottom of the module, there is a button that, when pressed, opens the console.

In order to solve the module, you must commit each file with the correct state in the correct order. You can commit files using the console with Git commands (Appendix G1t). Every commit must include exactly one file. After all files have been committed, you must push changes to the remote repository.
If you are done all correct, the module will solve; otherwise handle a strike, and show index of the first wrong committed file.
Note: After a strike, the module doesn't reset input. You must do it by yourself.

Initial file value

Take each character of SN, each first character of every file in number base 36 (1=1...A=10...Z=35), if that symbol is not a number and not a letter, count it as 13.
Sum it up and modulo 64.

File value

Look at all the conditions in the table on page 2. If the condition is true, take action in the column "Action". +, -, *, / are math operations for addition, subtraction, multiplication and division, respectively.

If the file value is less than zero, make him positive. That is the final file value (FFV). If the amount of files is even, sort them by his FFV in ascending order, otherwise in descending order. This is the order in which you must commit files. If you have the same FFV for two or more files, the first that appears on the module (going right) is preferred.

File status

Take FFV and multiply by the coefficient of initial status (Table Status coefficients). If a number has any decimal numbers, discard it. Let's name it as X. Then do a bitwise XOR between X and the multiplication of the repository coefficient and day of the week coefficient, modulo 2. This is the file's final status.

Repository coefficient

  • Sum up all the alphabetical positions of letter in the repository's name. (A=1, Z=26)
  • Modulo (amount of characters multiplied by 2).
  • Then add 1

Day of week coefficient

Use the day of the week on the module as a row, and your real day of the week (when the bomb was launched) as a column. The intersection will be your coefficient.

Appendix G1t

git add [file name]

Adding file. Setting status 1 when committed. Also, you can use file index instead of file name, where the first file that appears on the module is one.

git rm [file name]

Removing file. Setting status 0 when committed. Also, you can use file index instead of file name, where the first file that appears on the module is one.

git commit

Committing changes. Note: In one commit must be exactly one file. If in the commit be more than one file or zero, module will strike immediately.

git push

Pushing all commits to remote repository. While a command is input, the module will check for a solution. If your solution is correct, the module will be solved. Otherwise, the module will handle a strike and show the first wrong file index.

git status

Shows the current state of the repository. For example: if a commit has just occurred, since there is no difference between the previous commit and the current state of the repository, so the module will show the message "Commit is clear". If the previous command was git add/git rm, it will show the added file and his status.
